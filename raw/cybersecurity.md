# 网络安全与网络基础 (Cybersecurity & Networks)

网络安全旨在保护网络系统、硬件、软件和数据免受数字攻击、损坏或未经授权的访问。它是现代分布式系统和 Web 应用的重要保障。

## 1. 网络协议基石

网络通信基于分层架构。最经典的参考模型是 **TCP/IP 四层模型**（或 OSI 七层模型）。

*   **物理层 & 数据链路层**：负责物理介质（网线、光纤、Wi-Fi）上的数据帧传输（MAC 地址）。
*   **网络层 (IP)**：负责在不同网络之间路由数据包。使用 IP 地址定位设备。
*   **传输层 (TCP/UDP)**：
    *   **TCP (传输控制协议)**：面向连接的、可靠的、基于字节流的协议。通过三次握手建立连接，四次挥手释放连接，支持重传和流量控制。
    *   **UDP (用户数据报协议)**：无连接的、不可靠的、面向报文的协议。速度快，常用于视频直播、网络游戏等对实时性要求高的场景。
*   **应用层**：包含具体网络应用所使用的协议，如 HTTP、HTTPS、DNS、SSH。

## 2. Web 安全威胁 (OWASP Top 10)

Web 开发必须防范常见的客户端和服务器端攻击。

### XSS (跨站脚本攻击)
*   **原理**：攻击者在网页中注入恶意的客户端脚本（通常是 JavaScript）。当其他用户浏览该网页时，恶意脚本在用户的浏览器中运行，从而窃取用户的 Cookie/Session Token 或伪造用户操作。
*   **防御**：对用户输入进行严格的转义（HTML Entity Escaping），或者在 HTTP Header 中启用 CSP（内容安全策略）。

### CSRF (跨站请求伪造)
*   **原理**：攻击者诱导受害者访问第三方网站，利用受害者在目标网站的已登录状态（浏览器会自动携带目标网站的 Cookie），以受害者的名义发送恶意请求（如转账、修改密码）。
*   **防御**：使用 CSRF Token 校验、配置 Cookie 的 `SameSite=Strict` 属性。

### SQL 注入
*   **原理**：攻击者在输入框或 API 参数中拼接恶意的 SQL 语句，使数据库服务器执行未预期的查询或破坏操作。
*   **防御**：使用**参数化查询**（Prepared Statements），禁止直接拼接 SQL。

## 3. 密码学与安全防御 (Cryptography)

*   **对称加密**：加密和解密使用同一个密钥，速度快。代表算法：AES。
*   **非对称加密**：使用公钥加密，私钥解密；或私钥签名，公钥验签。代表算法：RSA、ECC。
*   **HTTPS 协议**：HTTP + SSL/TLS。在 TCP 握手之后，通过非对称加密安全地交换对称密钥，后续数据传输采用对称加密，防止数据在传输过程中被窃听或篡改。
*   **单向哈希函数 (Hash)**：将任意长度输入映射为固定长度输出，不可逆。常用于密码存储（必须加盐 Salt，防止彩虹表攻击）。代表算法：SHA-256、bcrypt。
