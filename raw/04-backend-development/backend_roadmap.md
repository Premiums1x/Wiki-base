# 后端开发学习路线与核心概念

后端开发关注的是服务器端逻辑、数据库交互、业务规则、API 设计以及系统架构。它支撑着前端展示所需的数据流。

## 1. 核心技术栈

### 运行环境与语言
*   **Node.js (JavaScript/TypeScript)**：使用 Express、NestJS 等框架，使前端开发者可以使用同一种语言进行全栈开发。异步非阻塞 I/O 使得它在处理高并发轻量级请求时表现优异。
*   **Go (Golang)**：Google 开发的静态编译型语言，具有极高的运行效率和极低的内存占用。其原生支持的轻量级线程（Goroutines）使得编写高并发后端服务非常简单。
*   **Java / Python / Rust**：其他主流后端选择，分别在企业级系统、人工智能/数据分析、高性能系统级开发中占主导地位。

---

## 2. 数据存储与管理

后端的核心职责之一是安全、高效地存储和检索数据。

### 关系型数据库 (SQL)
*   **代表**：MySQL, PostgreSQL
*   **特点**：使用表格结构，数据有严格的 Schema（模式）。支持 ACID 事务（原子性、一致性、隔离性、持久性），非常适合需要高度一致性的业务（如交易、财务系统）。
*   **核心概念**：外键关联、索引优化（B+树）、主从复制、SQL 查询联表（JOIN）。

### 非关系型数据库 (NoSQL)
*   **代表**：MongoDB (文档型), Redis (键值/缓存)
*   **特点**：Schema 更加灵活，易于水平扩展。
*   **Redis**：作为内存数据库，常被用作**缓存**（Cache）和分布式锁，大幅减轻关系型数据库的查询压力。

## 3. 接口协议设计 (APIs)

前端与后端进行通信的桥梁。

*   **RESTful API**：基于 HTTP 协议，使用标准的 HTTP 方法（GET, POST, PUT, DELETE）操作资源。返回数据格式通常为 JSON。
*   **GraphQL**：由 Facebook 开发的查询语言，允许客户端显式指定需要哪些数据，从而避免了 RESTful 接口中常见的“过度获取”（Over-fetching）或“获取不足”（Under-fetching）问题。
*   **gRPC**：基于 HTTP/2 和 Protocol Buffers，多用于微服务架构内部的高性能 RPC（远程过程调用）通信。
