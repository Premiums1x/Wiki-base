# DevOps 与云计算 (DevOps & Cloud Computing)

DevOps（Development 和 Operations 的组合）是一组过程、方法与系统的统称，用于促进开发、技术运营和质量保障部门之间的沟通、协作与整合。云计算则是实现 DevOps 自动化交付的基础基础设施。

## 1. 容器化与编排 (Containers & Orchestration)

容器技术彻底改变了软件交付的模式，解决了“在我的机器上能运行，但在服务器上不能”的历史难题。

### Docker
*   **概念**：将应用程序及其所有依赖项（库、配置文件等）打包成一个独立的**镜像**（Image），在沙箱化环境（Container）中运行。
*   **原理**：利用 Linux 内核的命名空间（Namespaces）进行资源隔离，利用控制组（Cgroups）进行资源限制，比传统虚拟机（VM）轻量得多，秒级启动。

### Kubernetes (K8s)
*   **概念**：工业级容器编排引擎，用于自动化容器化应用的部署、扩展和管理。
*   **核心组件**：
    *   **Pod**：K8s 中最小的调度单元，可以包含一个或多个容器。
    *   **Service**：定义一组 Pod 的访问策略，提供负载均衡和域名解析。
    *   **Deployment**：声明式地管理 Pod 的副本数量和滚动更新（Zero-downtime rolling update）。

## 2. 持续集成与持续交付 (CI/CD)

自动化流水线是 DevOps 的核心灵魂。

*   **持续集成 (CI)**：每次开发者提交代码（Git Push）后，自动触发代码构建、静态代码检查（Lint）、单元测试（Unit Test），确保新代码没有破坏已有功能。
*   **持续部署 (CD)**：测试通过的代码自动打包镜像，并发布到测试环境、预发环境甚至直接上线生产环境。
*   **常见工具**：GitHub Actions, Jenkins, GitLab CI.

---

## 3. 云计算架构与 IaC

### 云服务类型
*   **IaaS (基础设施即服务)**：提供计算虚拟化、存储和网络，如 AWS EC2, 阿里云 ECS。
*   **PaaS (平台即服务)**：提供开发和部署平台，隐藏底层服务器细节，如 Heroku, AWS Elastic Beanstalk。
*   **SaaS (软件即服务)**：通过浏览器直接使用的最终软件，如 Office 365, Notion。

### 基础设施即代码 (IaC)
*   使用配置文件来声明和管理云端的基础设施（如服务器、网络、防火墙），而不是手动在控制台点击创建。
*   **代表工具**：**Terraform**。通过声明式配置文件，一键在 AWS 或阿里云上创建整套网络和集群。
