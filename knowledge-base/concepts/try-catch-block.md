## 定义
异常捕获块（try-catch）是一种在编程语言中实现异常处理的结构。try-catch 块用于检测和处理程序执行过程中可能出现的异常情况，以保证程序的健壮性和可用性。它通常与抛出异常的技术 异常抛出机制 结合使用，共同构成了一套异常处理机制。

## 工作原理
try-catch 块的基本结构包括至少 `try` 子句和一个或多个 `catch` 子句。以下是对 try-catch 工作原理的详细解释：

* **try 子句**：在 try 子句中包含了可能产生异常或错误的代码。如果 try 块中的任意代码抛出了异常，则会立即中断 try 块中的执行并跳转到第一个匹配的 catch 子句。
* **catch 子句**：每个 catch 子句都有一个参数，通常称为捕获变量，用来接收从 try 子句抛出的异常对象。catch 子句用于处理异常，常见的处理方式包括记录错误信息、修复问题和恢复程序状态。

### 异常传播规则
系统会从 try-catch 块中抛出的异常传播至其外部的运行时环境，如果上层没有 try-catch 块，则异常将被静默忽略，这可能导致程序崩溃或行为异常。也有实现提供了顶级或全局性的 try-catch 机制来捕获所有未处理的异常。

### try-catch 与异步任务队列结合
在实现异步并发任务队列 async-task-queue 时，try-catch 块能保证当任务执行遭到错误拒绝时，整个任务调度系统不会因此崩溃。

在第【阶段二：架构重构与细节硬伤（版本 2）】中，我们见到了一个使用 try-catch 结构未能妥善捕获 Promise 拒绝事件的情况。事实上，组件应该采用异步的 Promise 捕获方式，即：
```js
try {
    await task();
} catch (err) {
    console.error('Task failed:', err);
}
```

这不仅可以妥善处理异常，同时还能保证执行流程的正确性及任务流的可靠性。

## 变体
try-catch 结构在不同的编程语言中有不同的实现方式。例如，在 JavaScript 中，可以通过 try-finally 结构来确保无论是成功执行还是抛出异常都能保证清理资源的代码被执行，提高程序的健壮性。

## 考虑因素
当使用 try-catch 来处理错误时，有几点需要考虑：
* **性能损失**：频繁的异常处理使程序复杂化，并可能带来一定的性能开销。
* **错误掩盖**：过度使用 try-catch 可能会掩盖程序中的根本性问题，而不是解决问题。
* **吞吐量优化**：在高吞吐量的系统中，静默忽略一些失败而允许系统继续运行可能是必要的，这也体现了高可用性上潜在的吞吐量优化。比如在异步任务队列中静默处理 promise 的 reject（error）以保证任务流的连贯性。
这些限制和考虑务必要结合具体使用场景来抉择是否采用 try-catch 机制。
