## Definition
Shitty First Draft is a principle practiced in creative and technical work, including programming, that encourages individuals to start working on a project by producing a suboptimal, incomplete first attempt without worrying about making it perfect. It is a strategy for overcoming paralysis caused by perfectionism and to establish a baseline for iteration and improvement. The imperfect-first-draft concept extends and underpins the philosophy that quality comes from refining and revising initial efforts rather than striving for flawlessness in the first draft.

## How it works
The principle leverages psychological resistance to imperfect initial output. When faced with a blank page or a daunting technical challenge, the urge to produce a near-perfect first draft can be overwhelming and discouraging. However, the Shitty First Draft technique advocates immediately starting work and focusing on creation over perfection, generating something that may initially be functional at best, and ugly and flawed at worst. Crucially, once the project has an initial version, the creator can iterate and refine it, often leading to a better end product.

In the context of software development, particularly for beginner-programmers or new interns, the methodology helps bypass the cognitive overload (commonly known as "analysis paralysis") and promotes actual early execution, which in turn provides instant feedback and learning opportunities. This resonates with the transformation from a student-mindset to an engineering-mindset, where practical action and iterative improvement are favored over meticulous planning.

### Variants
- **Minimum Viable Product (MVP) Mode**: Start with a basic, minimum viable version of a project, then add features and improve it over time. This mode is closely linked with the agile-development-methodologies, which build on the principle by emphasizing progressive refinement and user feedback.
- **First Useful Version (FUV)**: Focus on creating a version that serves the primary purpose of the software, even if it lacks bells and whistles. This aligns with the engineering mindset, as it prioritizes functionality before aesthetics and completeness.
- **Two-Minute Rule**: As detailed in the source material, this rule entails breaking down larger tasks into very small, manageable, almost trivial tasks; for example, just writing the first line of code or creating an HTML file. This optimization leverages the Shitty First Draft principle by reducing the psychological barrier to entry.

## Trade-offs
While the Shitty First Draft principle is powerful, it has inherent trade-offs. Key among these is the potential for discouragement due to poor early outputs, especially for perfectionists or those used to faster AI-generated flawless solutions. This dependence on AI, when transitioning to manual coding, can manifest as a fear of inadequacy and a tendency to stick with automated solutions that avoid the discomfort of imperfection.
