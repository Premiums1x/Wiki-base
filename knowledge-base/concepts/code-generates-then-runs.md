## Definition
Code Generates Then Runs refers to a development process where code is first created, either manually or through the use of tools like automated generation tools or AI, and then executed to produce the desired outcomes. This concept is crucial in understanding the workflow of programmers transitioning from a student-centric mindset to an engineering-centric approach in the AI era, as highlighted in ai-era-pain-points-and-mindset-transition. Specifically, it emphasizes the necessity of moving from the paralyzed state of analysis and decision-making to actually executing the code, a practice that [[shitty-first-draft]] principle extends and builds on.

## How it Works
In a traditional student-oriented approach, often referred to as "student thinking" or "exam thinking" student-thinking-vs-engineer-thinking, students tend to meticulously analyze and overthink problems before making any progress. This can lead to a situation known as "analysis paralysis" where the fear of making a mistake inhibits any action. A key aspect of the "Code Generates Then Runs" approach is to minimize the paralysis associated with planning by emphasizing action and execution at the earliest possible moment.

This shift involves breaking down complex tasks into smaller, more manageable parts, such as following the "Two-minute Rule" [[two-minute-rule]], which encourages developers to tackle even tiny actions that can be completed within a short timeframe. It also includes practices like writing a first rough version of the code, regardless of its quality, which fosters an experimental mindset and reduces the fear of failure ([[shitty-first-draft]]).

The process of generating code can involve a variety of methods, including manual coding, copy-pasting from existing implementations, or using AI-generated code snippets. In the latter, the role of a programmer shifts from that of a "bricklayer" to a "systems architect and director" who oversees the overall process and intervenes where necessary, working alongside AI as a "pair programming tutor" to ensure the solution fits the problem requirements and standards.

Execution, or running the generated code, is crucial as it provides immediate feedback on the code's functionality. This feedback loop is key to iterative improvement,凸显了学生思维 vs 工程思维转变的重要性。这一过程不仅缓解了因过度规划而导致的认知负担，还促进了动手实践的紧迫感。

## Variants
- **Manual Coding**: Here, the programmer writes code entirely by hand.
- **Automated Code Generation**: Tools and AI provide snippets or entire functions, reducing writing time while introducing challenges around understanding generated code.
- **Pair Programming with AI**: This method emphasizes interaction where the programmer is the primary creator and the AI serves as an assistant, suggesting solutions but not fully automating the task.
- **Incremental Development**: The approach focuses on developing the minimum viable product (MVP) before further refinement.

Each variant has its unique trade-offs related to development speed, code quality, maintainability, and programmer learning, respectively.

## Trade-offs
- **Quality vs Speed**: Manually writing code often leads to higher quality but takes more time compared to AI-generated code, which can be produced quickly but may contain more errors or be less aligned with project requirements.
- **Learning Experience vs Productivity**: Initial stages of development are often about learning and refining skills, whereas later stages prioritize productivity. Practices like the "Garbage First Draft" can significantly impact learning through hands-on experience despite leading to code that is initially less refined.
- **Dependency on AI**: The reliance on AI for solutions could erode essential coding skills and deepen feelings of inadequacy or "Imposter Syndrome" when left to code alone.
- **Feedback Loops**: Immediate execution and feedback are critical in code generation. The dependency on running code frequently can slow down processes if debugging and troubleshooting become overly time-consuming.

The balance between these aspects is crucial in fostering a productive and growth-oriented development environment, making the transition from student thinking to engineering thinking significantly impactful.
